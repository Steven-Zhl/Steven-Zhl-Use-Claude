# 需求完整性检查清单

在生成正式文档前，逐项确认以下内容。未确认项标注为【待定】，不阻塞文档生成，但需在"开放问题"章节记录。

## 业务层面

- [ ] 用户角色已明确，权限边界已描述
- [ ] 核心业务流程已梳理
- [ ] 状态流转已定义（如有）
- [ ] 不可逆操作已识别

## Collection设计层面

- [ ] 所有核心实体已识别
- [ ] 每个关联关系已判断：嵌入 or 引用，并有理由
- [ ] 最高频查询场景已明确（用于索引设计）
- [ ] 数据量级已评估
- [ ] 软删除策略已确认
- [ ] 通用字段已包含（_id, created_at, updated_at, is_deleted）

## API层面

- [ ] 所有功能点都有对应接口
- [ ] 分页参数已统一（page + page_size）
- [ ] 权限控制方式已明确
- [ ] 异常响应已定义
- [ ] 每个接口的DB操作已映射到封装方法

## 技术层面

- [ ] 外部依赖已列出
- [ ] 唯一性约束已识别（unique index）
- [ ] 需要聚合查询的场景已标注
- [ ] 定时任务或后台任务已说明（如有）

## 前端设计层面

- [ ] 所有核心页面的线框图已绘制
- [ ] 响应式适配策略已明确（移动端布局变化）
- [ ] 可复用组件已识别
- [ ] 主要交互操作已定义（点击/悬停/弹窗/跳转）
- [ ] 状态管理范围已界定（哪些状态放入 Pinia）
- [ ] 路由结构和权限控制已设计
